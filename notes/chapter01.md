# 什么是智能体

**定义**：智能体是能够通过传感器感知环境，并通过执行器采取行动以达成目标的自主实体。

四个基本要素：

- **环境**（Environment）：智能体所处的外部世界，如动态变化的道路交通、快速变化的金融市场。

- **传感器**（Sensors）：智能体通过传感器感知环境，例如：摄像头、麦克风、API 等。

- **执行器**（Actuators）：智能体通过执行器影响环境，例如：机械臂、方向盘、代码调用。

- **自主性**（Autonomy）：智能体基于感知和内部状态独立决策。

## 传统视角下的智能体

- **反射智能体**：完全依赖感知输入，条件-动作规则（如恒温器），无记忆。

- **基于模型的反射智能体**：“世界现在是什么样子的？”，拥有内部世界模型，具备初级记忆（如汽车的自动驾驶）。

- **基于目标的智能体**：“我应该做什么才能达成目标？”，主动规划以达成目标（如 GPS 导航）。

- **基于效用的智能体**：“哪种行为能为我带来最满意的结果？”，权衡多个目标，最大化期望效用。

- **学习型智能体**：不依赖预设，通过强化学习不断改进策略（如 AlphaGo Zero）。

## 大语言模型驱动的智能体新范式

**区别**：传统智能体依赖显式编程；LLM 智能体依赖预训练数据和涌现能力。

**特性**：

- 能处理模糊的自然语言指令。

- 内在规划与推理（例如，智能体会将高层级目标分解为一系列逻辑子任务）。

- 主动调用工具补全信息（例如，智能体识别到信息缺口，会主动调用外部工具来补全）。

- 根据反馈动态修正行为（例如，根据上下文动态修正自身行为的能力）。

## 智能体的类型

按决策架构：

- 反应式
  
- 模型式
  
- 目标式
  
- 效用式
  
- 学习型

按时间维度：

- 反应式：即时响应，速度快但短视。

- 规划式：深思熟虑，战略性强但耗时。

- 混合式：结合反应与规划。

按知识表示：

- 符号主义 AI：规则与逻辑，透明可解释。

- 亚符号主义 AI：神经网络，模式识别强但黑箱。

- 神经符号主义 AI：融合两者，兼具直觉与理性。

## 智能体的构成与运行原理

任务环境定义（PEAS 模型）：

Performance、Environment、Actuators、Sensors。

环境特性：

部分可观察、随机性、多智能体、序贯与动态。

运行机制（Agent Loop）：

感知 → 思考（规划+工具选择） → 行动 → 环境反馈 → 新观察 → 循环。

## 智能体应用的协作模式

作为开发者工具：辅助编程与工作流（如 GitHub Copilot、Claude Code、Cursor）。

作为自主协作者：独立完成高层级目标（如 AutoGPT、CrewAI、MetaGPT）。

Workflow vs Agent：

Workflow：预设流程，静态、确定。

Agent：自主推理，动态、灵活。

## 本章小结

智能体是具备自主性、能感知环境并采取行动的实体。

传统智能体到 LLM 智能体的演进，体现了从**规则驱动**到**数据驱动**的转变。

智能体的运行依赖 **感知-思考-行动-观察** 的闭环。

应用模式分为工具型与协作者型，Agent 更强调**自主性**与**目标导向**。
